**Findings**
- High: 管理员登录后端已存在，但前端没有对应登录流程。后端有 `POST /auth/admin/login`，见 [AuthController.java](/Users/mamba/dev/java_project/mini-/src/main/java/com/cug/miniblog/personalCenter/controller/AuthController.java:25) 和 [AuthServiceImpl.java](/Users/mamba/dev/java_project/mini-/src/main/java/com/cug/miniblog/personalCenter/service/impl/AuthServiceImpl.java:97)。前端登录页只调用普通登录 `userApi.login(...)`，见 [page.tsx](/Users/mamba/dev/java_project/mini-/frontend/app/login/page.tsx:47)。这意味着“管理员身份登录”目前没有前端入口。
- High: 管理员文章列表后端已存在，但前端没有对接。后端有 `GET /admin/articles`，见 [AdminArticleController.java](/Users/mamba/dev/java_project/mini-/src/main/java/com/cug/miniblog/contextManagement/controller/AdminArticleController.java:25)。前端 `articleApi` 只实现了 `getAdminDetail`，见 [api.ts](/Users/mamba/dev/java_project/mini-/frontend/lib/api.ts:225)，没有 `getAdminList`，也没有管理员文章管理页去查看“全站文章/跨用户文章/后台筛选结果”。
- Medium: 前端只“半接入”了管理员详情接口，而且接法偏普通用户流程，不是管理员后台。`/admin/articles/{id}` 被用于写作页加载编辑文章，见 [page.tsx](/Users/mamba/dev/java_project/mini-/frontend/app/write/page.tsx:76)，也被用于个人中心“我的文章”和“我的收藏”，见 [profile-tabs.tsx](/Users/mamba/dev/java_project/mini-/frontend/components/profile/profile-tabs.tsx:98) 和 [profile-tabs.tsx](/Users/mamba/dev/java_project/mini-/frontend/components/profile/profile-tabs.tsx:187)。这说明前端并没有单独实现管理员后台，而是把管理员详情接口当成通用详情接口在用。
- Medium: 前端没有按 `role` 做管理员功能分流。用户角色会存到状态里，见 [store.ts](/Users/mamba/dev/java_project/mini-/frontend/lib/store.ts:26)，但导航和入口没有基于 `user.role` 显示管理员菜单或后台入口，见 [header.tsx](/Users/mamba/dev/java_project/mini-/frontend/components/header.tsx:106)。当前 UI 只有“写文章 / 个人中心 / 发现文章”，没有“后台 / 管理 / 审核 / 置顶管理”等管理员入口。
- Medium: 管理员/运营型文章操作前端没有实现，例如置顶。后端暴露了 `PATCH /articles/{articleId}/top`，见 [ArticleController.java](/Users/mamba/dev/java_project/mini-/src/main/java/com/cug/miniblog/contextManagement/controller/ArticleController.java:73)。但前端文章 API 当前只做了创建、更新、删除，见 [api.ts](/Users/mamba/dev/java_project/mini-/frontend/lib/api.ts:230)，没有置顶切换接口，也没有对应按钮或管理页。

**结论**
当前后端“管理员相关功能”本身并不多，主要是：
- 管理员登录
- 后台文章列表
- 后台文章详情
- 一个更偏管理能力的置顶切换接口

前端的实际对接情况是：
- 已对接：后台文章详情 `GET /admin/articles/{id}`，但用途不是管理员后台，而是编辑页和个人中心数据补全。
- 未对接：管理员登录、后台文章列表、置顶管理、任何基于 `role` 的管理员入口或页面。
