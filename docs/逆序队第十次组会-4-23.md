# 逆序队第十次组会纪要

- 会议时间：2026 年 4 月 23 日
- 参会人员：陈侨、张战胜、董志鹏、张志彤

## 这次主要聊了什么

这次组会的重点已经从“主流程能不能跑”转到“功能能不能讲清楚、展示得顺不顺”上来了。前几天最明显的推进是标签相关能力基本立住了，不再只是页面里几个占位标签，而是后端有标签接口，文章列表能按标签筛选，前端也有标签页、文章详情标签跳转和发现页筛选。

除了标签线，大家也顺手把登录态和前端请求层过了一遍。因为后端已经开始接入令牌校验，如果前端还按原来的轻量方式发请求，后面越联调问题会越多，所以这次也把 token 注入、401 处理、接口调用方式一起理顺了。

## 每个人这几天推进到哪

- 陈侨这边主要把前端标签链路接起来了，包括发现页按标签筛选、文章详情页和文章卡片里的标签跳转，以及单独的标签页面。
- 张战胜把后端标签接口和文章按标签筛选这一段补齐了，标签不再只是展示字段，而是真正能参与文章查询。
- 董志鹏继续把登录态这条线往前推，补了令牌校验、登录拦截和前端请求里统一带 token 的处理方式。
- 张志彤这边主要帮着整理标签展示和页面呈现，把首页侧栏、详情页和发现页上的标签逻辑一起顺下来，也帮着从用户视角看筛选流程是不是自然。

## 联调时暴露了什么问题

- 标签之前只是前端展示层的静态内容，现在变成真实功能后，前后端字段命名、文章列表返回结构都需要一起调整。
- 前端请求层如果不统一处理 token 和 401，后面任何带登录态的页面都会出现“有时候能进、有时候突然失效”的问题。
- 标签功能接起来之后，旧数据库里那些“标签01、标签02”的占位数据就显得很别扭，演示时观感会很差，后面必须换成真实标签。

## 其他人给前端的建议

- 张战胜建议陈侨把标签点击路径做得更统一，不要首页能点、详情页不能点，或者能跳过去但筛选条件丢了。
- 董志鹏建议前端请求封装尽量收口，尤其是 401 这种全局问题，不要散在每个页面里各自处理。
- 张志彤建议标签页和发现页的展示文案再清楚一点，让老师一眼就能看出这是“按标签发现文章”，不是普通列表页。

## 下次组会前要完成什么

- 把数据库里的标签占位数据换成更具体的真实标签。
- 把文章内容和分类、标签关系再理顺，避免出现标签和文章主题不匹配的情况。
- 把个人中心和一些页面细节继续收一轮，减少演示时的突兀感。
