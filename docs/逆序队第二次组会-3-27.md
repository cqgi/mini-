# 逆序队第二次组会纪要

- 会议时间：2026 年 3 月 27 日
- 参会人员：陈侨、张战胜、董志鹏、张志彤

## 这次主要聊了什么

这次会议主要是把上次定下来的两条一级需求继续往下拆。大家不想后面出现“都知道要做，但没人知道自己具体要交什么”的情况，所以这次基本就是对着任务一个个往下落。

内容管理这边拆成了文章读操作、文章写操作、评论发布、评论删除、评论回复、分类管理。个人中心这边拆成注册、用户登录、管理员登录、获取资料、更新资料、获取我的文章、获取我的评论、收藏文章、取消收藏、获取收藏列表。私信功能和标签管理这次也提了，但最后一致意见还是先放后续迭代，不跟第一轮主功能抢时间。

## 每个人这几天推进到哪

- 陈侨把内容管理主线认得更细了，文章读写会分开处理，前端这边先按首页、详情、写作页、个人中心这几个入口去想页面。
- 张战胜把评论管理拆成几步，先做发布和查，再补删除和回复，点赞先不往前排。
- 董志鹏这边把注册、用户登录、管理员登录和资料管理串成一条线，后面可以一起测。
- 张志彤把收藏管理和个人中心列表能力接过去，尤其是“我的文章”“我的评论”“我的收藏”这三个列表的接口优先级比较高。

## 联调时暴露了什么问题

- 文章、评论、个人中心几个模块最后都会共用用户信息，这意味着公共实体最好尽早统一。
- 现在功能拆开以后，前端页面的优先级也得跟着排，不可能所有页面一起开。
- 标签管理虽然名字在需求里，但现在没有必要单独拉出一个前后台管理流程，先把文章核心流程做通更重要。

## 其他人给前端的建议

- 董志鹏建议前端先把登录、注册、个人资料这几个页面放在同一套交互风格里，字段和校验规则最好不要各写各的。
- 张志彤建议个人中心的结构一开始就按“资料 / 我的文章 / 我的评论 / 我的收藏”分区，不然后面页面肯定要返工。
- 张战胜建议文章详情页一定要给评论区留固定位置，后面评论发布、删除、回复才能顺着接进去。

## 下次组会前要完成什么

- 后端继续把评论模块和项目基础结构搭起来。
- 把个人中心和认证模块的接口再细化一下，保证每个人知道自己交什么。
- 陈侨这边开始整理前端的页面入口和数据流，不急着写满，但至少把路线图先画出来。
