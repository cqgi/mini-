# 逆序队第三次组会纪要

- 会议时间：2026 年 3 月 30 日
- 参会人员：陈侨、张战胜、董志鹏、张志彤

## 这次主要聊了什么

这次会议更多是在看后端基础框架有没有站稳。张战胜那边评论模块已经开始有雏形了，数据库驱动、MyBatis-Plus、评论相关的数据类和接口也在往上搭。大家顺着这个进度，把项目结构又对了一遍，避免后面每个人的包结构越写越散。

另一个重点是页面和接口风格的问题。陈侨这边虽然前端还没大面积开写，但已经开始把页面入口、API 封装方式和状态管理先想了。因为我们已经能看出来，后端不同模块的返回风格很可能不完全一样，这件事如果前期不提前兜住，后面前端会一直被接口细节拖着跑。

## 每个人这几天推进到哪

- 陈侨这边先把前端要用到的页面入口、组件边界和接口清单记下来了，准备等接口再稳定一点就开始搭页面。
- 张战胜已经把评论模块的基础框架、发布评论和查看评论的思路顺下来了，点赞功能先记着，优先级没那么高。
- 董志鹏这边开始往认证和个人资料那条线上走，准备把用户注册、登录和资料管理尽快补出来。
- 张志彤在梳理个人中心需要依赖的文章、评论、收藏数据关系，后面这部分很吃后端聚合接口。

## 联调时暴露了什么问题

- 评论模块虽然在往前推，但最开始还没挂上完整的 Web 测试环境，说明框架搭起来和真正跑通还是两回事。
- 项目结构在慢慢统一，意味着之前写得比较散的地方后面可能还要改包名、改引用。
- 前端现在已经能预感到一个问题：如果接口前缀、返回结构、字段命名不统一，后面每接一个页面都要单独适配一次。

## 其他人给前端的建议

- 董志鹏建议陈侨别在页面里直接写请求细节，最好先做一层统一请求封装，不然认证模块和内容模块一混，很快就看不清。
- 张志彤提醒个人中心的数据最好别写死，后面“我的文章”“我的评论”“我的收藏”会是三种不同列表结构，前端要早点考虑复用。
- 张战胜建议评论组件至少把“一级评论 + 回复列表”的结构预留出来，哪怕第一版样式先简单一点。

## 下次组会前要完成什么

- 评论模块继续往“能实际跑起来”推进。
- 认证、资料、文章相关接口开始实做。
- 前端准备进入页面骨架和请求层设计阶段，先把最核心的路径留出来。
